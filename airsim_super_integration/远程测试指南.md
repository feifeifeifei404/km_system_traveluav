# 远程SSH连接测试SUPER指南

本指南适用于**通过SSH远程连接服务器**进行SUPER测试的场景。

---

## 🔑 关键点

**AirSim必须在服务器本地（有显示器）启动，其他组件可以在远程SSH终端运行。**

---

## 📋 测试流程

### 准备工作

1. **确认配置文件已修复**
   ```bash
   # 远程SSH终端执行
   cat ~/Documents/AirSim/settings.json | grep ApiServerPort
   ```
   应该看到：`"ApiServerPort": 41451,`

2. **停止旧的AirSim进程**（如果有）
   ```bash
   # 远程SSH终端执行
   cd /mnt/data/airsim_super_integration/scripts
   ./stop_airsim.sh
   ```

---

## 🚀 启动步骤

### 步骤1: 在服务器本地启动AirSim 🖥️

**⚠️ 此步骤必须在服务器本地（有显示器的机器）执行！**

```bash
cd /mnt/data/TravelUAV/envs/closeloop_envs
./ModularPark.sh -ResX=1280 -ResY=720 -windowed
```

**等待30-40秒**，直到看到：
- ✅ AirSim窗口出现
- ✅ 无人机模型显示
- ✅ 场景完全加载

---

### 步骤2: 在远程SSH终端验证 💻

```bash
# 检查AirSim进程
ps aux | grep ModularPark | grep -v grep

# 检查API端口（最重要！）
netstat -tuln | grep 41451
```

**必须看到类似输出：**
```
tcp    0    0 0.0.0.0:41451    0.0.0.0:*    LISTEN
```

如果端口**没有监听**：
1. 在服务器本地重启AirSim
2. 确认`settings.json`配置正确
3. 等待更长时间（可能需要60秒）

---

### 步骤3: 启动SUPER规划器（远程SSH）

**远程SSH终端 - 终端1：**
```bash
conda deactivate
cd /mnt/data/airsim_super_integration/scripts
./start_super_airsim.sh
```

等待看到SUPER启动成功的消息。

---

### 步骤4: 启动桥接节点（远程SSH）

**远程SSH终端 - 终端2：**
```bash

**成功的标志：**
```

**如果还是连接失败：**
1. 确认步骤2中端口41451正在监听
2. 确认AirSim场景完全加载（在服务器本地检查）
3. 等待更长时间后重试

### 步骤5: 启动RViz（远程SSH - 可选）

## 🐛 常见问题

### 问题1: 桥接节点提示 "ECONNREFUSED"

**原因**：AirSim的API服务器未启动

**解决方案**：
1. 在远程终端检查端口：
   ```bash
   netstat -tuln | grep 41451
   ```

2. 如果端口没有监听：
   - 在服务器本地重启AirSim
   - 确认`settings.json`有`"ApiServerPort": 41451,`
   - 等待场景完全加载（至少30秒）

### 问题2: 无法启动RViz窗口

**原因**：远程SSH没有图形界面转发

**解决方案**：

**方法1：使用X11转发**
```bash
# 重新SSH连接时加上-X参数
ssh -X user@server
```

**方法2：使用VNC或远程桌面**
- 使用VNC/RDP连接服务器图形界面
- 在VNC会话中启动RViz

**方法3：不启动RViz**
- RViz只是可视化工具，不是必需的
- 可以通过ros2命令行检查话题数据

### 问题3: 找不到settings.json

## 💡 最佳实践

1. **总是先验证API端口**：`netstat -tuln | grep 41451`
2. **给足够的启动时间**：AirSim至少等30秒，SUPER等5秒
3. **按顺序启动**：AirSim → SUPER → 桥接节点 → 发送目标
4. **保持终端窗口**：不要关闭正在运行组件的终端
5. **使用tmux或screen**：方便管理多个SSH会话

### 使用tmux管理会话（推荐）

```bash
# 安装tmux（如果没有）
sudo apt install tmux

# 创建新会话
tmux new -s super_test

# 创建多个窗口
# Ctrl+B, C (创建新窗口)
# Ctrl+B, N (切换到下一个窗口)
# Ctrl+B, P (切换到上一个窗口)
# Ctrl+B, 0-9 (切换到指定窗口)

# 分离会话（保持运行）
# Ctrl+B, D

# 重新连接会话
tmux attach -t super_test
```
