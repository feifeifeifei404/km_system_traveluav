# TravelUAV 场景连接测试

## 问题诊断

您遇到的场景加载失败问题是由于 **msgpack-rpc 传输过程中的字节串类型不匹配** 导致的：

### 问题原因

1. **服务器端问题**：接收的场景名称是字节串 `b'Carla_Town01'`，无法在字典中匹配字符串键 `'Carla_Town01'`
2. **客户端问题**：服务器返回的 IP 是字节串 `b'127.0.0.1'`，导致 `str(b'127.0.0.1')` 变成 `"b'127.0.0.1'"` 而不是 `'127.0.0.1'`，断言失败

### 已修复

已在以下文件中添加了字节串到字符串的转换：

1. **`airsim_plugin/AirVLNSimulatorServerTool.py`**
   - 在 `_open_scenes()` 函数开始处转换场景名称

2. **`airsim_plugin/AirVLNSimulatorClientTool.py`** 
   - 在接收服务器返回结果后转换 IP 地址

## 常见问题

### Q: 服务器未运行
如果看到 "场景服务器未运行" 错误，请先启动服务器（见步骤1）

### Q: 连接超时
- 检查防火墙是否阻止 25000 端口
- 确认服务器正在监听 127.0.0.1:25000

### Q: 场景文件未找到
- 确认 `/mnt/data/TravelUAV/envs/carla_town_envs/Town01/LinuxNoEditor/CarlaUE4.sh` 存在
- 检查文件权限是否可执行

## 技术细节

### 字节串转换位置

**服务器端** (`AirVLNSimulatorServerTool.py:460`):
```python
def _open_scenes(self, ip: str , scen_id_gpu_list: list):
    # 将所有字节串转换为字符串
    scen_id_gpu_list = [
        (scen_id.decode('utf-8') if isinstance(scen_id, bytes) else scen_id, gpu_id)
        for scen_id, gpu_id in scen_id_gpu_list
    ]
    ...
```

**客户端** (`AirVLNSimulatorClientTool.py:200`):
```python
ip = result[1][0]
# 将字节串转换为字符串
if isinstance(ip, bytes):
    ip = ip.decode('utf-8')
```
